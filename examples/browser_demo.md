# 自動操作瀏覽器實戰 (Automated Browser Operations)

Antigravity Agent 不僅僅侷限於 IDE 內部，它還能啟動並控制瀏覽器，協助你完成與 Web 相關的任務。

## 什麼情況下適合使用？

- **驗證響應式設計 (RWD)**：請 Agent 調整視窗大小，確認網頁在手機版與桌面版是否都能正常顯示。
- **除錯前端錯誤**：當按鈕沒反應時，請 Agent查看瀏覽器 Console 是否有 JavaScript 報錯或網路請求失敗 (404/500)。
- **複雜流程測試**：繁瑣的像是「註冊 -> 收信驗證 -> 登入 -> 購物車 -> 結帳」這類跨頁面流程，交給 Agent 自動跑一遍。
- **文件與解法搜尋**：當遇到未知的錯誤代碼，讓 Agent 直接去 GitHub Issue 或 Stack Overflow 找類似災情的解法。
- **即時資料驗證**：檢查 Deploy 後的網站是否正確顯示了最新的資料庫內容，無需人工逐一比對。

## 如何使用？

只需在對話中明確下達指令，例如：
- 「打開瀏覽器搜尋 React 19 的新特性」
- 「幫我測試登入頁面的錯誤處理流程」
- 「去這個 URL 抓取表格資料」

Agent 會啟動一個受控的瀏覽器實例（你會看到它打開），並將操作過程錄製下來。**操作結束後，錄影檔 (WebP) 會自動儲存在 Artifacts 列表中。**

## 人機協作與介入 (Human Intervention)

雖然是自動化，但你隨時可以取回控制權。瀏覽器視窗是完全互動的，你可以在 Agent 操作的空檔隨時點擊或輸入。

- **驗證碼 (CAPTCHA) 與 2FA**：遇到時，你可以手動輸入，再叫 Agent 繼續。
- **關鍵決策**：在送出付款或刪除資料前，你可能希望親自確認最後一眼。

## 安全處理帳號密碼 (Secure Credential Handling)

若流程中包含登入，務必注意安全性：

- **❌ 錯誤做法**：直接在對話框輸入「我的密碼是 xxxx」或寫在程式碼中。
- **✅ 推薦做法**：使用 `.env` 檔案或環境變數，讓 Agent 讀取變數來登入。

## 處理複雜流程的指引技巧 (Guiding Complex Workflows)

當網站流程太過複雜，請嘗試：

1.  **拆解步驟 (Step-by-Step)**：明確指令 1, 2, 3 步驟。
2.  **建立指引文件 (Checklist)**：寫好 `login_flow.md` 讓 Agent 照著做。
3.  **視覺化暗示 (Visual Hints)**：描述按鈕的外觀或位置（如「紅色警告下方的按鈕」）。

## ⚠️ 目前的限制 (Limitations)

Agent 「無法」操作瀏覽器以外的視窗 (OS Level)：
- **檔案上傳/下載視窗**：Agent 看不到。請手動選擇檔案，或改用 Drag & Drop / API 上傳。
- **列印對話框**：Agent 無法控制。請改用「截圖」或「Save as PDF」指令。
- **硬體驗證**：指紋、YubiKey 等需人工操作。
